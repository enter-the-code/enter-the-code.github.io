실습 설명


3번째 수업은 udp 소켓으로 통신하는 실습을 했다. tcp 소켓은 물총마냥 어떤 통에 물이 있으면 그걸 다 발사 할때까지 계속 트리거를 땡기고있면 통에 물이 차례로 줄면서 통이 비어간다 이걸 통신으로 보면 tcp는 따로 통신에 쓰는 버퍼가 있어서 거기서 자료를 다 받아올때까지 원하는 만큼 끊어서 받아올수 있다는 것이다 udp는 총이다. 그냥 딱 한발 쏘고나면 끝이다. 그 한발에 모든 공격력을 담아 보내는 것이다. 따라서 통신으로 보면 버퍼에 담아서 원하는 만큼 받아오는게 아니라 특정 정보를 다 먹고 싶으면 그만큼 크기를 키워서 한번에 받아와야 된다는 것이다. 따라서 받아 올 버퍼 크기를 충분히 설정해야 모든 정보를 담아올수있는것이 udp이고 많은 버퍼 크기를 쪼개서 원하는 만큼 만 따로 전부 소켓버퍼에서 꺼내올수 있는것이 tcp이다 추가로 udp는 빠르지만 불안정하고(소실) tcp는 안전하다.

int isStringDouble(char *s)
{
    size_t size = strlen(s);
    if (size == 0)
        return 0; // 0바이트 문자열은 숫자가 아님

    for (int i = 0; i < (int)size; i++)
    {
        if (s[i] == '.' || s[i] == '-' || s[i] == '+')
            continue;
        if (s[i] < '0' || s[i] > '9')
            return 0; // 알파벳 등이 있으면 숫자 아님

}
